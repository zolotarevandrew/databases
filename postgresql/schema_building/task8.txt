Описание: База данных для управления рестораном.
Требования:

Столы имеют ID и вместимость.
Блюда имеют ID, название и цену.
Заказы привязаны к столу, включают блюда, количество и время заказа.
Задание:

Спроектируйте три отношения и нормализуйте их до третьей нормальной формы (3NF).
Добавьте возможность учёта скидок на определённые блюда.


dishes
- id int;
- name text;

dish_prices
- id int;
- dish_id int;
- price decimal;
- date date;

цена блюда на дату.

dish_sales
- id int;
- dish_id int;
- date date;
- percent int;

скидки блюда на дату.

tables
- id int;
- name text;
- capacity int; 

здесь пока что нет смысла в поле capacity, потому что нет кол-ва гостей, а привязывать к кол-ву блюд странно,
на одном столе с capacity 3, может поместиться хоть 25 блюд.

orders
- id int;
- table_id int;
- date datetime;

заказ определяется столом и датой.

order_dishes
- id int;
- order_id int;
- dish_id int;
- quantity int;

блюда в заказе определяются заказом и кол-вом.
